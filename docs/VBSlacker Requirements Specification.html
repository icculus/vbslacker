Objective

VBSlacker is a suite of development tools that allow users to develop cross-platform applications using the standard Visual Basic® language.

Overview

The VBSlacker development suite is intended to bring the Rapid Application Develop technology to non-Microsoft® community. Although VBSlacker will be fully compatible with WIN32, its real goal is to bring the Visual Basic language to the rest of the world, while still being source-code compatible with existing Visual Basic applications.

 

Section 1 - Compiler

The VBSlacker compiler will convert Visual Basic source code into C code, particularly 'gcc' compliant code. Although it should keep to ANSI standard as much as possible.

The compiler needs to support (Visual Basic Project) for compiling and making Visual Basic Projects. It should interpret the project and compile the modules that are defined in the project. Below is a list of Visual Basic modules that need to be supported by VBSlacker…

.VBP

Visual Basic Project

.FRM

Form

.FRX

Binary data associated with a Form

.BAS

Module

.CTL

User Control

.CTX

Binary data associated with a User Control

Section 2 - Install Wizard

The Install Wizard needs to provide the following list of functions…

    • The ability to install the VBSlacker suite under multiple operating systems. Perhaps the install program itself could be a Java application.
    • To create install programs for applications developed under VBSlacker.
    • Installs should have the option to contain multiple binaries (for different operating systems).
    • Needs to be as easy to use as Visual Basic's Application Setup Wizard.
